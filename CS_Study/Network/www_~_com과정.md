# 주소창에 www.~.com을 입력했을 때의 과정

![주소창에URI](/img/web/주소창에URL치면.png)
출처: http://tcpschool.com/webbasic/works

웹 브라우저 주소창에 www.naver.com을 입력하면
1. 서버로 가기전에 캐싱을 한다. 캐시(cache)는 받아온 데이터를 저장하는 저장소이고, 캐싱은 저장한 리소스중 만료되지 않은 리소스를 제공하는것
   - 브라우저 캐시
     - 쿠키와 로컬스토리지등으로 브라우저가 HTTP를 통해 받아온 리소스를 보유한다
   - 프록시 캐시
     - 사용자들과 인터넷망 사이에 위치한 프록시 서버를 통해 서버로 부터 받아온 데이터를 캐싱
2. 캐시에서 가져올 데이터가 없다면 DNS 서버에게 요청할 서버의 IP주소를 받아온다
   - DNS는 도메인 이름과 IP주소를 맵핑해주는 서버
3. 인터넷 통신
   - TCP/IP layer, 라우팅을 통해 실제 서버를 찾아간다
4. 초기 연결
   - 3way handshake, SSL연결등을 통해 연결을 성정 후 요청 / 응답
5. 응답한 데이터를 브라우저에서 렌더링
